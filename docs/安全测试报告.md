## 已修复问题

| 风险 | 问题 | 影响范围 | 修复 |
| --- | --- | --- | --- |
| 中 | 缺少统一基础安全响应头 | API 和上传静态资源响应 | 新增 `server/middlewares/security.js`，设置 `X-Content-Type-Options`、`X-Frame-Options`、`Referrer-Policy`、`Permissions-Policy`、基础 CSP |
| 中 | 生产环境 CORS 可被配置为 `*` | API 跨域访问 | 生产环境不接受 `CORS_ORIGIN=*` 通配 |
| 中 | 高风险写接口缺少统一轻量限流 | 登录、验证码、上传、社区、预约、活动报名、订单写操作 | 新增内存限流中间件并接入关键路由 |
| 中 | 500 错误在开发模式可能返回内部错误消息 | API 错误响应 | 500 统一返回“服务器内部错误”，内部错误仅写服务端日志 |
| 高 | 文件上传允许危险中间扩展名风险 | 上传文件名校验 | `a.php.jpg` 这类危险双扩展名现在会被拒绝 |

## 自动化安全测试

脚本：`scripts/security-smoke.js`

运行：

```bash
node scripts/security-smoke.js
```

覆盖：

- 未登录访问用户接口失败
- 普通用户访问后台失败
- 用户不能查看他人订单
- 用户不能取消他人预约
- 用户不能通过 profile payload 修改他人资料
- SQL 注入 payload 不绕过登录、不打崩公开搜索
- 前端无危险 HTML 渲染 API
- `.php` / `.js` / `.html` / `.svg` / 双扩展名上传失败
- 路径穿越删除上传文件失败
- 被限制预约用户直接调用预约接口失败
- 被限制发帖用户直接调用发帖接口失败
- 维护中座位不可预约
- 满员活动不可继续报名
- 已取消订单不可非法完成
- 后台导出接口普通用户不可访问
- `.env` 不被静态访问
- 错误响应不暴露 stack
- 基础安全头存在
- 非白名单 CORS Origin 被拒绝
- 关键操作写入 `audit_logs`
